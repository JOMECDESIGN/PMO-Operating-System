## 它如何映射到 CLAUDE.md

宪法本是为华翔智能座舱 demo 写的(M1–M7 / 杭州·昆山 / F1 CAS·F2 接口)。本项目按 `README.md`"适配到其他项目"做三处替换:

| 宪法构件 | demo 版 | **本项目版** |
|---|---|---|
| F1 纵向冻结 | CAS / A 面数据 | **LED 异形屏尺寸冻结(7/10)+ 需求/边界冻结(6/26)** |
| F2 横向冻结 | 接口契约(两站) | **接口矩阵 / 接口责任矩阵(附录H)**(本项目从两站升级为**多供应商**,F2 更重) |
| 里程碑 | M1–M7 周制 | **8 阶段日历倒排**;唯一外部硬日期 11/10 替代车展 |

## 一句话定调

本项目**出厂验收前全程零浮动**——整条都是关键路径。所以 PMO 的全部杠杆压在**两道不可顺延冻结线(6/26 需求冻结、7/10 LED 下单)** 和 **把珠海进场的不可控性前置消化在乙方现场(现场作业最小化)** 这两件事上。按宪法"缓冲耗尽不得报绿",**项目整体健康基线即为 AMBER**。
