# 三道缝(S1 / S2 / S3)— 二代太空体验舱

> 宪法 §2 的本项目落点。每条 PMO 动作都应能追溯到一道缝。本项目约 70% 延期风险在缝里,不在任一团队能力上。

## S1 — 供给缝(双供给源)

本项目供给缝有**两个独立来源**,治理方式不同:

| 来源 | 供给内容 | 可控性 | 治理 |
|---|---|---|---|
| **航天五院(甲方)** | 8 个甲方确认点 · A3 科学性参数审定 · A4/A8 决策 | 可协调 | 甲方确认点日历(见 `milestones.md`);逾期等额顺延后续节点 |
| **航天组委会(外部)** | 珠海进场窗口(时间、吊装、供电报备) | **不可控** | "现场作业最小化"原则;**建议 9 月前锁定进场窗口**;吊装/供电方案提前报备 |

> **PMO 关键动作**:把组委会这个不可控外部源,通过"现场作业最小化 + 提前锁窗口"转成可管理节点。对应风险 **R8**。

## S2 — 跨场地 / 多供应商缝

本项目是**多场地 + 多供应商**汇流,比 demo 的两站更复杂:

- **场地链**:乙方集成场地 ↔ **昆山工厂(震动预调,9/25)** ↔ 珠海现场 ↔ 常驻运营地。
- **两次转场**(出厂→珠海、珠海→常驻地)= 全生命周期**风险最集中动作**。拆装五类损耗:模组损伤 / 接插件松动 / 结构偏差 / 亮色度变化 / 校正参数丢失。
- **多供应商汇流**:LED 屏体 · 座椅骨架 · 震动模块 · 健康设备 · 播控/音视频 — 各家在乙方现场集成时汇合。

**治理**:
1. 迁移专项(方案 14.4):拆装编号 / 专用包装减震 / 复装整屏复校(参数云端备份)/ 按调试矩阵回归。对应 **R3**。
2. **接口责任矩阵(附录H)**:划清多供应商交付边界,防"联调反复、责任不清"。对应 **R7**。
3. 工厂预联调机制(45°CAVE 9/25)把跨供应商问题前置暴露。

## S3 — 团队间缝(关键路径)

本项目**整条都是关键路径**,团队间缝集中在并行建设的收敛点:

- **阶段3 设备生产 ∥ 阶段4 软件+内容**(均 7/27–9/18)→ **9/14 现场整舱集成处收敛**。
- 跨专业依赖:内容 ← 分屏模板/图像融合;软件 ← 接口矩阵;震动参数 ← 样机/工厂预调(A1)。
- 14 类角色(项目经理、技术负责人、舱体/座舱域/LED/前端/后端/UI、内容/三维/音频、测试、现场实施、运维培训)按阶段动态投入。

**治理**:并行成立的前提是 **6/26 需求冻结会 + 接口矩阵冻结**;任一不冻 → 并行发散 → 10/20 现场联调爆炸(正是宪法 S2 警告场景)。关键路径周度追踪缓冲消耗。
