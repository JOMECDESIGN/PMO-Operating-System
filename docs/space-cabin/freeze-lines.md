# 两道冻结线(F1 / F2)+ 变更控制 — 二代太空体验舱

> 宪法 §3 的本项目落点。冻结后,任何改动须走:请求 → 影响评估 → 审批 → 更新基线。无口头、无侧信道。
> 文档明确 **6/26 与 7/10 为"不可顺延"两节点** —— 它们就是本项目的两条冻结线。

## F1 — 物理 / 范围冻结(纵向级联)

F1 含两道子冻结:

| 子线 | 冻什么 | 何时 | 危险形态 |
|---|---|---|---|
| **F1a 需求/边界冻结** | 需求范围 · 接口 · 验收口径 · 现场复测结论 | **6/26 需求冻结会** | 范围漂移 → 并行无依据 |
| **F1b LED 物理冻结** | LED 异形屏尺寸 · 结构 · 排产 → **下单** | **≤7/10** | **纵向级联**:尺寸→钢结构→安装→图像融合/分屏内容适配。下单后改=全链返工 |

> **7/10 LED 下单是全项目单点杠杆最高的事件**:它既是 F1 纵向冻结,又是关键路径起点。错过 → 10/31 出厂验收不保。

## F2 — 接口契约冻结(横向 / 跨供应商)

| 冻什么 | 何时 | 危险形态 |
|---|---|---|
| 接口矩阵(方案 6.5 / 附录H)· 接口责任矩阵 · 三端数据字段 | 6/26 起,**7/24 深化设计评审锁定** | **横向**:多供应商 + 阶段3∥4 并行;一方改而他方不知 → 10/20 现场联调爆炸 |

> 本项目 F2 比 demo 更重:从"两站对接"升级为"**多供应商对接**"。F2 是阶段3(生产)∥ 阶段4(软件/内容)并行成立的前提。

## 五步变更流程(两线通用)

1. **提交** — 书面变更请求:内容、原因、命中哪条冻结线、紧急度。无口头变更。
2. **登记分类** — PMO 记入变更追踪;分 次要 / 重大 / 跨供应商(决定审批路径)。
3. **评估+审批** — 按类型(见下)。
4. **执行同步** — 执行方落地;PMO 确保**所有受影响方(尤其多供应商)** 被通知并更新。
5. **更新基线** — PMO 更新数据版本 / 接口矩阵版本;归档审批。关闭。

### 路径 3.1 — F1 物理/范围变更(纵向)
按影响范围路由至 舱体/结构、座舱域、LED、内容 各负责人评估;涉甲方审定项(造型/科学性)加五院节点;后 PM 终批。**LED 下单后的尺寸变更视为重大变更,必须评估对钢结构/安装/分屏内容的级联工期。**

### 路径 3.2 — F2 接口契约变更(横向)
**多方并行会签**:请求同时流至受影响的各供应商 + 后端/内容负责人;任一异议即退回。批准后各方同步更新代码/接口/分屏模板;PMO 更新接口矩阵版本并广播"接口 vX 已变更,各方同步"。

## 质量闸

每次冻结内容变更须有完整记录(请求/评估/审批/执行/基线更新)。**一次接口变更,只有在"所有受影响供应商已同步"确认后才能关闭** —— 这是防现场联调爆炸的最后一道闸。对应风险 R2 / R7。

## 不可顺延说明(来自方案 15.3.4)

> 甲方确认点逾期将等额顺延后续节点;**其中 6/26 与 7/10(LED 下单前提)为不可顺延节点,逾期将无法保障 10 月底出厂验收交付**。
